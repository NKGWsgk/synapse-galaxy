# Synapse Galaxy プロジェクト仕様書

## 1. プロジェクト概要

**コンセプト:** 大人の知的好奇心を可視化し、コンテンツ同士を「なぜ繋がるか」という自由記述で結ぶシナプス型SNS。  
4×4のコンパスグリッドは、中心に置いたコンテンツとそこから伸びる接続を探索するためのビューポートとして機能する。

**対応デバイス:** PCのみ（スマホ対応は将来検討）

## 2. 技術スタック

- **Frontend:** Next.js (App Router), Tailwind CSS, Framer Motion
- **Backend/DB:** Supabase (PostgreSQL, Auth)
- **AI:** Gemini API（自由記述からのキーワード抽出）
- **Auth:** Google OAuth（Supabase Auth経由）

## 3. データベース設計

#### Canonicalization（名寄せ）

登録時に Gemini を使って「この URL が過去登録済みのどの作品と同一か」を判定し、同一なら同じ canonical_id を付与する。

## 4. キーワードと接続の太さ

- 投稿時に Gemini が description からキーワードを抽出し `keywords[]` に保存
- 同一の source_url ↔ target_url ペアにおいて、複数のシナプスが同じキーワードを持つほど、UI上の接続線を太く表示する
- キーワードはコンパスUI上でラベルとして浮遊表示する

## 5. アクセス制御・閲覧ゲート

### 5.1 閲覧制限

| 状態 | 制限 |
|------|------|
| 未ログイン | コンパス探索N回 かつ シナプス閲覧M件まで（暫定: 探索5回・閲覧10件） |
| ログイン済み・投稿9件以下 | 探索・閲覧に一部制限あり（暫定値） |
| シナプスを10件以上投稿済み | 制限解除（全件閲覧・探索し放題） |

「書いたら見れる」設計。件数はすべて調整可能。

### 5.2 認証

- Google OAuth（Supabase Auth経由）
- 登録・審査なし、誰でもGoogleアカウントでログイン可能

## 6. いいね機能

- 1ユーザー1シナプスにつき1いいねのみ（重複不可）
- いいね数は `synapses.likes_count` に非正規化して保持（集計コスト削減）

**いいねできる場所：**
1. 周囲の小さいOGPカード（ホバーで表示）
2. コンパス中心クリック → 詳細パネル内
3. 接続線上のキーワードラベルをクリック → 接続の自由記述ポップアップ → そこでいいね

## 7. ユーザーランキング・プロフィール

- 受け取ったいいね数の多いユーザーをランキング表示
- ランキングからそのユーザーの `/user/:id` ページへ遷移可能
- `/user/:id` ではそのユーザーが繋いだシナプス一覧を表示
- ランキングの表示場所は保留（コンパスUIがメイン、他要素は後で決める）

## 8. UI/UX：画面レイアウト

### 8.1 全体構造

```
[ ヘッダー: 検索バー | 投稿ボタン | ログイン/ユーザーアイコン ]
[                                                              ]
[                  コンパス（画面いっぱい）                      ]
[                                                              ]
```

- コンパスをメインに据えて画面を最大限使う
- 投稿フォームはモーダルで開く（常時表示しない）
- ユーザーランキングの表示場所は後で決める

### 8.2 コンパスUI

- **中心2×2マス:** フォーカスコンテンツ（OGP画像・タイトル表示）
- **周囲12マス:** 接続先コンテンツのOGPカード
- フォーカスコンテンツをクリック → 詳細パネル（OGP・接続一覧・接続の理由・いいねボタン）
- 周囲のカードをクリック → そのコンテンツがフォーカス中心に移動（探索）
- OGPカードにホバー → いいねボタン表示

### 8.3 接続線とキーワード

- 中心と周囲カードを結ぶ線の太さは、共有キーワード数に応じて変化
- キーワードラベルを線の中点付近に浮遊表示
- キーワードラベルをクリック → 接続の自由記述ポップアップ → いいねできる

**同一URLペアに複数シナプスあり得る：**
複数人が同じ2コンテンツを別々の理由で繋げることができる。キーワードが被るほど線が太くなり「見えない繋がり」が浮き彫りになる。

### 8.4 初期表示

- トップページを開いたとき、フォーカス中心は**人気コンテンツ**（いいね数・シナプス接続数の多いもの）

### 8.5 グリッド外周の表示

4×4グリッドの外側にも接続の気配を表示し、1画面で完結する。

- グリッドの各辺から外側へシナプス線が伸びる（接続がさらに先へ続くことを示す）
- 線の上にキーワードラベルを表示
- グリッド＋外周のシナプス線＋キーワードすべてが1画面に収まるレイアウト

### 8.6 検索

- ヘッダーの検索バーでURL or タイトルを入力
- 選んだコンテンツがコンパスの中心にセットされる

## 9. 通知機能

- ヘッダーにベルアイコンを設置、未読数をバッジ表示
- アプリ内通知のみ（メール通知なし）

**通知トリガー：**
1. 自分のシナプスにいいねがついた
2. 自分が投稿したシナプスの起点/接続先コンテンツに、他のユーザーが新しいシナプスを繋いだ

### DB: `notifications`

| カラム | 型 | 説明 |
|--------|-----|------|
| id | UUID | Primary Key |
| user_id | UUID | 通知を受け取るユーザー |
| type | Text | `liked` / `new_synapse` |
| synapse_id | UUID | 対象シナプス |
| actor_id | UUID | アクションしたユーザー |
| read | Boolean | 既読フラグ |
| created_at | Timestamptz | |

## 10. ページ構成

| パス | 内容 |
|------|------|
| `/` | メインのコンパスUI |
| `/user/:id` | ユーザーのプロフィール・シナプス一覧 |

## 10. Smart Input（投稿フォーム）

1. 起点URL・接続先URL を入力 → OGPを自動取得しプレビュー表示
2. 接続の理由（自由記述）を入力
3. 送信 → Gemini がキーワード抽出 → DB保存

※ カテゴリ選択は廃止。

## 11. アフィリエイトリンク変換

ユーザーが投稿したURLは、可能な限りアフィリエイトリンクに自動変換して保存する。

| サービス | 変換方法 |
|---------|---------|
| Amazon.co.jp / Amazon.com | URLに `tag=（アフィリエイトID）` パラメータを付与（実装済み） |
| 楽天 | （今後対応予定） |
| その他 | 対応サービスが増えたら随時追加 |

- 変換はサーバーサイド（API Route）で行い、変換済みURLをDBに保存
- ユーザーには変換を意識させない（UI上は通常のURLとして表示）

## 13. 実装ロードマップ

1. **DB整備** — category_id 削除・synapse_likes / notifications テーブル追加
2. **認証** — Google OAuth 設定
3. **UIリファクタ** — ヘッダー（検索・投稿ボタン・ログイン・ベルアイコン）+ コンパス全画面化 + 投稿フォームのモーダル化
4. **閲覧ゲート** — 未ログイン/ログイン/投稿数に応じたアクセス制御
5. **検索** — コンテンツ検索 → コンパス中心セット・初期表示を人気コンテンツに
6. **いいね機能** — API・UI（OGPカードホバー・詳細パネル・キーワードポップアップ）
7. **通知機能** — ベルアイコン・未読バッジ・通知一覧
8. **ユーザーページ** — `/user/:id` プロフィール・シナプス一覧
9. **ユーザーランキング** — いいね集計・表示
10. **接続線の太さ** — キーワード共有数に応じた可変太さ表示
11. **テキスト表示調整** — 各マスのタイトル・キーワードラベルの文字サイズ・文字数制限を見やすく調整
